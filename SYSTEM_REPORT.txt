1. THE IDEA
================================================================================

Product Name    : AI Lead Intelligence Platform (LeadIntel)
Tagline         : "Know Who to Call Before You Dial"
Product Type    : B2B SaaS — Sales Intelligence & Lead Prioritization
Target Market   : SMB to Mid-Market Sales Teams (5-500 reps)
Pricing Model   : Subscription SaaS (Freemium → Pro → Enterprise)

PRIMARY OUTCOMES:
  • Increase sales-qualified-lead (SQL) conversion rate by 30-50%
  • Reduce time-to-first-contact by 60%
  • Eliminate manual lead research — AI does it automatically
  • Personalized outreach at scale with AI-drafted emails

CORE VALUE PROPOSITIONS:
  VP-1 : Reps spend 100% of their time on leads most likely to convert
  VP-2 : AI-generated explanations tell reps WHY a lead is hot
  VP-3 : Real-time buying signals (funding, hiring, tech adoption) surface
         automatically — no manual research required
  VP-4 : Personalized AI-drafted email templates per lead
  VP-5 : Model performance transparency builds trust with sales leadership
  VP-6 : Seamless CRM-ready export — integrates with Salesforce, HubSpot

HOW IT WORKS:
  1. Sales team imports leads (CSV upload or manual entry)
  2. System automatically enriches each lead using free APIs:
     - Company data from website scraping & DuckDuckGo
     - Email validation via DNS MX record checks
     - News signals from NewsAPI (funding, hiring, growth)
  3. AI scoring engine rates each lead 0-100 across 5 weighted categories
  4. Leads are classified as Hot (>=70) / Warm (40-69) / Cold (<40)
  5. Google Gemini AI generates personalized insights & email drafts
  6. Sales reps see a prioritized dashboard — call the hottest leads first

BUSINESS MODEL:
  Free Tier  : 100 leads/month, basic scoring
  Pro Tier   : 2,000 leads/month, AI emails, signals, API access
  Enterprise : Unlimited leads, custom models, CRM integrations

3. SYSTEM ARCHITECTURE — WAT FRAMEWORK
================================================================================

The entire system is built on the WAT Framework (Workflows, Agents, Tools):

  ┌──────────────────────────────────────────────────────────────────┐
  │                     LAYER 1: WORKFLOWS                          │
  │  Markdown SOPs defining WHAT to do and WHEN                     │
  │  ┌──────────────────┐ ┌────────────────┐ ┌──────────────────┐   │
  │  │ lead_enrichment  │ │ lead_scoring   │ │ email_generation │   │
  │  │      .md         │ │     .md        │ │      .md         │   │
  │  └──────────────────┘ └────────────────┘ └──────────────────┘   │
  └──────────────────────────────┬───────────────────────────────────┘
                                 │
                        Workflows define steps
                                 │
  ┌──────────────────────────────▼───────────────────────────────────┐
  │                     LAYER 2: AGENTS                             │
  │  Python orchestrators that READ workflows and COORDINATE tools  │
  │  ┌──────────────────────────────────────────────────────────┐   │
  │  │           LeadEnrichmentAgent                            │   │
  │  │  • enrich_lead()  → runs enrichment pipeline             │   │
  │  │  • score_lead()   → runs scoring pipeline                │   │
  │  │  • generate_ai_content() → runs AI generation            │   │
  │  └──────────────────────────────────────────────────────────┘   │
  └──────────────────────────────┬───────────────────────────────────┘
                                 │
                      Agents call deterministic tools
                                 │
  ┌──────────────────────────────▼───────────────────────────────────┐
  │                     LAYER 3: TOOLS                              │
  │  Python scripts that DO the actual work — deterministic & fast  │
  │                                                                 │
  │  ┌─────────────────┐  ┌─────────────────┐  ┌────────────────┐  │
  │  │ domain_extractor│  │ email_validation│  │  web_scraper   │  │
  │  │   (free)        │  │   (free/MX)     │  │   (free)       │  │
  │  └─────────────────┘  └─────────────────┘  └────────────────┘  │
  │  ┌─────────────────┐  ┌─────────────────┐  ┌────────────────┐  │
  │  │    news_api     │  │  ai_generator   │  │  lead_scorer   │  │
  │  │ (free 100/day)  │  │ (Gemini free)   │  │  (rule-based)  │  │
  │  └─────────────────┘  └─────────────────┘  └────────────────┘  │
  └──────────────────────────────────────────────────────────────────┘

WHY WAT FRAMEWORK:
  • Separates AI reasoning (agents) from deterministic execution (tools)
  • If a tool fails, the agent retries or skips — doesn't crash the pipeline
  • Workflows are human-readable markdown — non-engineers can understand them
  • Self-improving: when errors happen, workflows get updated with fixes
  • Each layer can be tested independently

     - Activity log: called, emailed, meeting_booked, qualified, converted

     - Error handling: last_error, dead_letter support

  10. api_keys
      - Customer API key management for external integrations

SECURITY:

================================================================================

MODEL: Rule-Based v1.0 (designed to upgrade to XGBoost ML later)

SCORING FORMULA (0-100 total):

  ┌────────────────────────┬────────┬──────────────────────────────────┐
  │ Category               │ Weight │ What It Measures                 │
  ├────────────────────────┼────────┼──────────────────────────────────┤
  │ Firmographic Fit       │  30%   │ Company size, funding, growth,   │
  │                        │        │ industry match (SaaS/tech)       │
  ├────────────────────────┼────────┼──────────────────────────────────┤
  │ Person-Level Fit       │  20%   │ Seniority (C-Suite=10, VP=8),    │
  │                        │        │ decision maker, email quality    │
  ├────────────────────────┼────────┼──────────────────────────────────┤
  │ Signal Strength        │  30%   │ Weighted sum of buying signals   │
  │                        │        │ (funding=1.5x, hiring=1.2x)     │
  ├────────────────────────┼────────┼──────────────────────────────────┤
  │ Data Completeness      │  10%   │ % of enrichment fields filled    │
  ├────────────────────────┼────────┼──────────────────────────────────┤
  │ Source Quality          │  10%   │ Referral=10, Demo=9, Organic=7,  │
  │                        │        │ Paid=5, Import=3, Cold=2         │
  └────────────────────────┴────────┴──────────────────────────────────┘

TIER CLASSIFICATION:
  🔥 Hot  : Score >= 70  (call these first!)
  🌡  Warm : Score 40-69  (nurture and monitor)
  ❄  Cold : Score < 40   (low priority)

EXPLAINABILITY:
  • Pseudo-SHAP values show % contribution of each scoring category
  • Feature importance breakdown displayed on lead detail page
  • Score history chart tracks changes over time

UPGRADE PATH:
  • When 1,000+ labeled leads are available → train XGBoost model
  • SHAP library ready for real feature importance
  • A/B test: rule-based vs ML model conversion rates

10. ENRICHMENT PIPELINE FLOW
================================================================================

When a new lead is created (manual or CSV import), the following happens:

  TRIGGER: Lead created with auto_enrich=true
       │
       ▼
  ┌─ STEP 1: Extract Domain ──────────────────────────────────────────┐
  │  Tool: domain_extractor.py                                        │
  │  From email (skip gmail/yahoo) or company_website URL             │
  └───────────────────────────────────────────────────┬───────────────┘
                                                      │
  ┌─ STEP 2: Scrape Company Website ──────────────────▼───────────────┐
  │  Tool: web_scraper.py → scrape_company_website()                  │
  │  Extract: title (company name), meta description                  │
  └───────────────────────────────────────────────────┬───────────────┘
                                                      │
  ┌─ STEP 3: Search Company Info ─────────────────────▼───────────────┐
  │  Tool: web_scraper.py → search_company_info()                     │
  │  DuckDuckGo Instant Answer API (free, unlimited)                  │
  └───────────────────────────────────────────────────┬───────────────┘
                                                      │
  ┌─ STEP 4: Validate Email ──────────────────────────▼───────────────┐
  │  Tool: email_validation.py                                        │
  │  Check: format regex + DNS MX record lookup                       │
  │  Output: email_verified, email_deliverable, is_professional       │
  └───────────────────────────────────────────────────┬───────────────┘
                                                      │
  ┌─ STEP 5: Detect Buying Signals ───────────────────▼───────────────┐
  │  Tool: news_api.py                                                │
  │  Search company in NewsAPI (100 free requests/day)                │
  │  Detect: FUNDING, HIRING, EXECUTIVE, EXPANSION signals           │
  │  Each signal gets a strength score (1-10)                         │
  └───────────────────────────────────────────────────┬───────────────┘
                                                      │
  ┌─ STEP 6: Calculate Completeness ──────────────────▼───────────────┐
  │  Count filled fields / total fields × 100                         │
  └───────────────────────────────────────────────────┬───────────────┘
                                                      │
  ┌─ STEP 7: Save & Update Status ────────────────────▼───────────────┐
  │  Upsert enrichment data → lead_enrichment table                   │
  │  Insert signals → lead_signals table                              │
  │  Update lead → enrichment_status = 'complete'                     │
  └───────────────────────────────────────────────────┬───────────────┘
                                                      │
  ┌─ STEP 8: Run Scoring Pipeline ────────────────────▼───────────────┐
  │  Automatically triggers score_lead()                              │
  │  Calculates 0-100 score across 5 categories                       │
  │  Assigns Hot/Warm/Cold tier                                       │
  │  If score >= 50: triggers AI content generation                   │
  │    → Google Gemini generates insight + email draft                │
  └───────────────────────────────────────────────────────────────────┘
